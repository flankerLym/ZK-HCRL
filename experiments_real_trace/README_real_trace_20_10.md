# Real-trace 20/10 experiment for TCO-DRL HCRL

This folder is a parallel experiment folder. It does **not** modify the original baseline experiment folders. It adds a real-trace-driven runner that trains on the first 20 days of `real_oracle_trace.csv` and evaluates on the last 10 days.

## Data split

Input trace: `experiments_real_trace/data/real_oracle_trace.csv`

Default split:

- Train: timestamps from the beginning of the trace to `start + 20 days`
- Test: timestamps after `start + 20 days`

For your current 30-day trace, this gives roughly:

- Train: about 8640 requests
- Test: about 4280 requests

The runner uses the trace rows as realistic oracle-risk contexts. The request arrival process is still generated by the original workload model, so the scheduling pressure remains comparable to the synthetic experiments.

## What the real trace drives

For each request, the trace supplies:

- service type from `service_type`
- oracle/reference price deviation from `deviation`
- data freshness / staleness from `staleness`
- threshold-derived validation risk from `validation_success`
- threshold-derived anomaly level from `anomaly_label`

The code does **not** treat `anomalous` as a confirmed real attack. It uses the label as a risk signal to create a realistic trace-driven scheduling environment.

## Quick smoke test

Run this first:

```powershell
.\experiments_real_trace\scripts\smoke_test_real_trace.ps1
```

## HCRL-only train/test

```powershell
.\experiments_real_trace\scripts\run_real_trace_20_10_hcrl.ps1 `
  -TraceCsv "experiments_real_trace\data\real_oracle_trace.csv" `
  -Epoch 30 `
  -Seed 3 `
  -OraclesPerType 40 `
  -MaliciousRatio 0.30
```

Outputs:

- Training outputs: `experiments_real_trace/output/train_hcrl/`
- Test outputs: `experiments_real_trace/output/test_hcrl/`

## RL baselines + HCRL train/test

This trains and tests DQN, RA-DDQN, PB-SafeDQN, COBRA-Oracle, and HCRL-Oracle using the same 20/10 split.

## Notes

- The original baseline experiment folders are not touched.
- The runner imports the root project's `model.py`, `env.py`, `utils.py`, and `param_parser.py`.
- The new runner is `experiments_real_trace/real_trace_main.py`.
- The real-trace adapter is `experiments_real_trace/real_trace_env.py`.
- The test command uses `--Eval_Only --Greedy_Eval` and loads the weights saved by the train run.
- If you want multi-seed paper results, run the script with `-Seed 3`, `-Seed 4`, `-Seed 5`, then summarize the output CSVs.
